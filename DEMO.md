# GitGrade Demo Guide

## Quick Demo Script

### 1. Start the Application
```bash
npm run dev
```

### 2. Demo Repositories to Try

#### High-Quality Repository (Expected: 80-95/100)
```
https://github.com/vercel/next.js
```
**Why**: Well-documented, comprehensive tests, active development, production-ready

#### Medium-Quality Repository (Expected: 60-75/100)
```
https://github.com/expressjs/express
```
**Why**: Good structure, some documentation, active but could improve testing

#### Learning Project (Expected: 40-60/100)
```
https://github.com/your-username/my-first-project
```
**Why**: Basic structure, minimal documentation, limited tests

### 3. Demo Flow

1. **Enter Repository URL**
   - Paste one of the demo repositories
   - Click "Analyze"

2. **Show Loading State**
   - Explain the analysis stages
   - Mention it takes 5-10 seconds

3. **Review Results - Overview Tab**
   - **Score Card**: Highlight the overall score and grade
   - **AI Summary**: Read the summary, show strengths/weaknesses
   - **Timeline**: Show commit activity over time
   - **Skill Map**: Highlight detected vs missing skills
   - **Improvement Simulation**: Show potential score improvement

4. **Score Breakdown Tab**
   - Click on each dimension to expand
   - Show explainable scoring
   - Highlight why points were lost/gained
   - Show detected signals

5. **Roadmap Tab**
   - Show prioritized improvement steps
   - Explain the "Why" and "How" for each item
   - Show estimated impact scores

6. **Recruiter Lens Tab**
   - Show hireability score
   - Explain level assessment
   - Highlight red flags and green flags
   - Read the verdict

### 4. Key Features to Highlight

#### ⭐ Recruiter Lens (Unique Feature)
- "This is what makes GitGrade special"
- Show how it evaluates from a recruiter's perspective
- Explain the level assessment

#### ⭐ Explainable AI Scoring
- Click on any dimension
- Show the detailed breakdown
- Explain how signals are detected

#### ⭐ Improvement Simulation
- "If you follow the roadmap..."
- Show predicted score improvement
- Highlight level upgrade potential

### 5. Error Handling Demo

Try these to show graceful error handling:
- Invalid URL: `https://github.com/invalid/repo`
- Private repository (if you have access)
- Non-existent repository

### 6. Technical Highlights

- **Fast Analysis**: 5-10 seconds for most repositories
- **Comprehensive**: 6 dimensions, 100 points total
- **AI-Powered**: Personalized summaries and roadmaps
- **Production-Ready**: Error handling, graceful degradation
- **Beautiful UI**: Modern, responsive design

## Presentation Tips

1. **Start with a problem**: "How do recruiters evaluate GitHub repos?"
2. **Show the solution**: GitGrade provides comprehensive, explainable analysis
3. **Demo the unique features**: Recruiter Lens, Explainable Scoring, Improvement Simulation
4. **End with impact**: "This helps students improve their GitHub presence"

## Sample Talking Points

- "GitGrade doesn't just give you a score - it tells you WHY and HOW to improve"
- "The Recruiter Lens feature is unique - it shows exactly what recruiters see"
- "Every score is explainable - click any dimension to see why points were lost"
- "The improvement simulation shows you the potential impact of following the roadmap"
- "Built for production with proper error handling and graceful degradation"
